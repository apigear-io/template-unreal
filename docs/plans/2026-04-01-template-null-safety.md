## Task 1: Add parameter count validation in OLink server invoke (SA-07)

**Files:**
- Modify: `templates/module/Source/moduleolink/Private/Generated/OLink/olinksource.cpp.tpl:121-134`

**Step 1: Add `args.size()` validation before parameter extraction**

The current pattern at lines 121-134 extracts parameters without checking the array:

```
{{- range .Interface.Operations }}
	if (path == "{{.Name}}")
	{
{{- range  $i, $e := .Params }}
		{{ueType "" .}} {{ueVar "" .}} = args.at({{ $i }}).get<{{ueReturn "" .}}>();
{{- end }}
```

Replace the operation block (lines 121-134) with a version that validates array size before extraction:

```
{{- range .Interface.Operations }}
	if (path == "{{.Name}}")
	{
{{- if len .Params }}
		if (!args.is_array() || args.size() < {{ len .Params }})
		{
			UE_LOG(Log{{$Class}}, Error, TEXT("olinkInvoke: '{{.Name}}' expects {{ len .Params }} arg(s), got %d"), args.is_array() ? static_cast<int>(args.size()) : -1);
			return nlohmann::json();
		}
{{- end }}
{{- range  $i, $e := .Params }}
		{{ueType "" .}} {{ueVar "" .}} = args.at({{ $i }}).get<{{ueReturn "" .}}>();
{{- end }}
```

Key points:
- Only emit the check when the operation has parameters (`if len .Params`)
- Zero-param operations skip the check (no args to validate)
- Returns empty json on validation failure rather than executing with default-initialized params
- `static_cast<int>` on `args.size()` for clean `%d` formatting

## Task 3: Add `checkf` fallback guards for Shipping builds (SA-14)

**Files:**
- Modify: `templates/module/Source/modulemonitor/Private/Generated/Monitor/loggingdecorator.cpp.tpl:60-84`
- Modify: `templates/module/Source/modulemsgbus/Private/Generated/MsgBus/msgbusadapter.cpp.tpl:141-159`
- Modify: `templates/module/Source/moduleolink/Private/Generated/OLink/olinkadapter.cpp.tpl` (same pattern)

All three files have identical `setBackendService`/`_setBackendService` patterns with `checkf` as the only null protection. In Shipping builds, `checkf` compiles out.

**Step 1: Add graceful fallback after each `checkf` in loggingdecorator.cpp.tpl**

For each `checkf` in `setBackendService` (lines 66, 71, 77), add a matching `if (!ptr) return;` after it:

Find (lines 65-67):
```
		U{{$Iface}}Publisher* BackendPublisher = BackendService->_GetPublisher();
		checkf(BackendPublisher, TEXT("Cannot unsubscribe from delegates from backend service {{$Iface}}"));
		BackendPublisher->Unsubscribe(TWeakInterfacePtr<I{{$Iface}}SubscriberInterface>(this));
```

Replace with:
```
		U{{$Iface}}Publisher* BackendPublisher = BackendService->_GetPublisher();
		checkf(BackendPublisher, TEXT("Cannot unsubscribe from delegates from backend service {{$Iface}}"));
		if (!BackendPublisher)
		{
			return;
		}
		BackendPublisher->Unsubscribe(TWeakInterfacePtr<I{{$Iface}}SubscriberInterface>(this));
```

Apply the same pattern to line 71 (`checkf` on `InService.GetInterface()`) and line 77 (`checkf` on subscribe publisher).

**Step 2: Apply the same pattern in msgbusadapter.cpp.tpl**

Same `checkf` pattern at lines 147, 152, 157. Add `if (!ptr) return;` after each.

**Step 3: Apply the same pattern in olinkadapter.cpp.tpl**

Same `checkf` pattern at lines 60, 65, 71. Add `if (!ptr) return;` after each.
